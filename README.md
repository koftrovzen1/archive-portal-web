# archive-portal-web

## 项目简介

`archive-portal-web` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对特定域名或网站，而是作为静态 HTML 页面集合的存储与展示入口，方便对历史页面进行统一管理和访问。

## 目录结构

```
archive-portal-web/
├── index.html          # 入口页面，提供归档页面的导航或索引
├── pages/              # 存放各个独立 HTML 页面归档
│   ├── example-page/   # 示例归档目录（可按需命名）
│   │   └── index.html
│   └── ...
├── assets/             # 公共资源文件（CSS、JS、图片等）
│   ├── styles/
│   ├── scripts/
│   └── images/
└── README.md           # 本文件
```

## 页面归档说明

- 每个独立页面归档应放置在 `pages/` 目录下，建议使用有意义的目录名。
- 页面内容应保持完整，包括 HTML、CSS、JavaScript 及相关资源文件。
- 如需引用公共资源，请通过相对路径指向 `assets/` 目录。
- 建议在每个归档目录中包含 `index.html` 作为默认入口。

## 维护说明

- 新增页面归档时，请在 `pages/` 下创建新目录并放置完整页面文件。
- 如需更新现有归档，请直接替换对应文件，并保持目录结构不变。
- 建议在提交信息中简要说明归档的页面来源或更新原因。
- 请勿在页面内容中包含任何敏感信息或违反相关法律法规的材料。
